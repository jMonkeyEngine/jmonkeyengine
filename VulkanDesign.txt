
This is the core idea: all objects that interact with native memory (NIO buffers, vulkan buffers, etc.) are simply memory addresses. A memory address in this context holds a) the address of the first byte pointed to and b) the size of the addressed region. Importantly, memory addresses contain a method that emits a similar address but offset by a specific number of bytes. Some memory addresses are relative to another memory address, and thus defer address queries to the parent address.

So take a struct bound to a buffer. When a field within the struct needs to interact with the buffer's memory, it queries for the address of its struct, which in turn queries for the address of its buffer. The returned address object is opaque.
